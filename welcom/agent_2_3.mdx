## マルチエージェント概要

左側ナビゲーションの **「エージェント」** パネルから、ご自身のエージェントライブラリを閲覧できます。
&#x20;その中から必要なエージェントを選択し、中央のキャンバスエリアへドラッグ＆ドロップしてください。
&#x20;こうして配置されたエージェントは **「サブエージェント（子エージェント）」** として機能し、柔軟でパワフルなマルチエージェント構成が実現可能になります。

## キーコンポーネントとルール

**サブエージェントの構成における基本ルールは以下の通りです：**

1. &#x20;コアモジュール：&#x20;

   * [基础模块](https://uxkpl4cba3j.feishu.cn/wiki/Eiz2wzZGniKsBIkaboCcpyngn5g)（基本モジュール）

   * [Agent对话结束模块](https://uxkpl4cba3j.feishu.cn/wiki/Q97AwZD6SiHNF2kJbX5c1Ffnnsg)（Agent終了モジュール）

2. 重要な制限：各サブエージェントには **「Agent対話終了」モジュールは1つのみ**設置可能です。

**親エージェントと子エージェントの相互作用:**

&#x20;これらの状態フローのメカニズムを理解するには、以下の表を参照してください。

| パラメータ名                         | タイプ    | 機能説明                                               |
| ------------------------------ | ------ | -------------------------------------------------- |
| **子エージェント実行完了後、ユーザーに次ステップを確認** | -      | 表示されるポップアップの質問文を設定します                              |
| **親エージェントの再実行**                | -      | 子エージェント完了後、「最初から再実行」ボタンで親Agent全体を初期状態からやり直します      |
| **満足 → 親エージェント継続**             | -      | 子エージェント実行完了後、以降のユーザー入力はこのエージェントに再渡されず、次のモジュールへ進みます |
| **不満 → 子エージェント再実行**            | -      | 親エージェント全体ではなく、該当子エージェントのみを再実行します                   |
| **確認ポップアップを表示**                | ON/OFF | ON：子エージェント実行後に確認ポップアップ表示OFF：ユーザーへの表示なしで処理が自動進行     |
