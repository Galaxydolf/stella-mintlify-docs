## ドキュメント入力（アップロードした文書への質問応答）

![](https://uxkpl4cba3j.feishu.cn/space/api/box/stream/download/asynccode/?code=Njc5NjgwOGVjZThiZmY0OWUwZThjMDQwNDE5N2NmYTdfc0xDU2RkUWVuZzFJcVVGQ0Q0S2Y5MDU4dGJqR3dTOEtfVG9rZW46QjV6eGJ6SHBjbzUwZHF4b0VaZmNNanNwbmVmXzE3NDI5OTc0Mjg6MTc0MzAwMTAyOF9WNA)

**モジュール概要**

#### ユーザーがドキュメントをアップロードし、質問を入力すると、AIモデルが文書を解析し、回答を生成する。（例： 「この文書の要点をまとめてください」の場合、即時にドキュメント内容をもとにした回答を提供する）

「ドキュメントレビュー」モジュールとの違い：本モジュールでは、ユーザーが自由に質問を入力可能（ 「ドキュメントレビュー」モジュールは、あらかじめ設定された質問に基づいて回答を生成するため、ユーザーが個別に質問を入力する必要がない）

**モジュールパラメータ**

| パラメータ名            | データ形式        | 機能説明                                                                                                                                                        |
| ----------------- | ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **連動アクティベーション**   | ブール型         | 本ノードは複数のブール型入力を接続可能。**全ての条件が `true`&#x20;**&#x20;の場合に本モジュールを実行（例：　「すべての条件が満たされた場合に実行」 ）                                                                     |
| **任意条件アクティベーション** | ブール型         | 本ノードは複数のブール型入力を接続可能。**いずれか1つでも条件が `true`&#x20;**&#x306E;場合に本モジュールを実行（例：　「いずれか1つでも条件を満した場合に実行」 ）                                                             |
| **情報入力**          | 文字列型         | 前処理の出力テキストをAIモデルに入力する                                                                                                                                       |
| **ドキュメント情報**      | 文字列型         | ・ 「ユーザー入力」モジュールの「ドキュメント情報」から出力されたデータを受け取る・使用には「ユーザー入力」モジュールの「ドキュメントアップロード」を有効にし、入力フィールド右側にアップロードボタンを追加してください（**対応フォーマット：PDF、DOC、DOCX、TXT（最大50MB、最大9ファイル））** |
| **モデル選択**         | -            | ・本モジュールで使用する AI**モデル** を選択可能・異なるモジュールごとに、異なるAIモデルを設定可能                                                                                                      |
| **システムプロンプト**     | -            | AIモデルに対して、処理すべきタスクを**自然言語（Prompt）**&#x3067;指示                                                                                                               |
| **ユーザーへの返信表示**    | -            | オフ時、モジュール処理は実行されるが、ユーザーチャット画面には表示されない（例：内部処理のみでユーザーに直接返信する必要がない場合等）                                                                                         |
| **返信のクリエイティブ度**   | -            | ・AI回答の自由度が調整可能・数値が小さいほど厳密な回答となり、大きいほど創造的な回答となる                                                                                                              |
| **回答文字数制限**       | -            | 回答の最大文字数を制限する（使用AIモデルによって異なる）                                                                                                                               |
| **応答終了**          | ブール型         | すべての処理が完了すると `はい / いいえ` を出力し、次のコンポーネントの実行条件として利用可能                                                                                                          |
| **応答内容**          | ナレッジベース検索結果型 | AIが生成した回答を出力。次のモジュールの入力データとして使用することも可能                                                                                                                      |
| **モジュール実行完了**     | ブール型         | **本モジュールの実行完了時、出力値は `True` となる**（通常、次処理を開始するためのトリガーとして使用）                                                                                                   |
