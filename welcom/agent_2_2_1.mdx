## **ユーザー質問（クエリ受付）**

<img
  src="/images/18.png"
/>


**モジュール概要**

* AIシステムにおいて、ユーザーのクエリを正確に取得することは、ユーザーの意図解析の重要なプロセスです。本モジュールは、ユーザーが入力した質問や選択内容を取得し、それを他のモジュールに連携・送信する役割を担います。

**モジュールパラメータ概要**

| パラメータ名            | データ形式 | 機能説明                                                                                                                                                                                                        |
| ----------------- | ----- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **連動アクティベーション**   | ブール型  | 本ノードは複数のブール型入力を接続可能。**全ての条件が `true`&#x20;**&#x20;の場合に本モジュールを実行（例：　「すべての条件が満たされた場合に実行」 ）                                                                                                                     |
| **任意条件アクティベーション** | ブール型  | 本ノードは複数のブール型入力を接続可能。**いずれか1つでも条件が `true`&#x20;**&#x306E;場合に本モジュールを実行（例：　「いずれか1つでも条件を満した場合に実行」 ）                                                                                                             |
| **テキスト入力**        | 文字列型  | ユーザーが自由にテキストを入力できる機能。入力内容は「テキスト情報」ノードを通じシステムに送信（デフォルトで有効）                                                                                                                                                   |
| **ドキュメントアップロード**  | 文字列型  | ユーザーがドキュメントをアップロードしての質問が可能。「ドキュメント質問」モジュールと組み合わせて使用し、「ドキュメント情報」ノードを通じて出力が可能となる有効後、入力フィールド左側にドキュメントアップロードボタンが追加される最大9個のファイルを同時アップロード可能、各ファイルの最大サイズ：**50MB****対応フォーマット：pdf 、 doc 、docx、txt**                  |
| **画像アップロード**      | 文字列型  | ユーザーが写真をアップロードして質問できるようにします。通常は「画像質問」モジュールと組み合わせて使用され、「画像情報」ノードを通じて出力されます。オンにすると、入力ボックスの左側に写真をアップロードするためのエントリが追加されます。最大 9 つのファイルを同時にアップロードできます (各ファイルのサイズは 30 MB を超えることはできません)。 PNG、JPG、JPEG 形式がサポートされています。 |
| **ドキュメントレビュー**    | 文字列型  | 有効後、「ドキュメントレビュー」モジュールと組み合わせて使用し、「ドキュメント比較の有無」の設定が可能となる入力フィールド右側に**ドキュメントアップロードボタン**が追加され、設定した数のドキュメントをアップロードし、比較が可能となる                                                                                      |
| **カスタムボタン追加**     | -     | 機能の利用シーン任意のアクション実行ノードとして使用可能。（例：特定の処理を実行する、ナレッジベースを検索する）入力結果のフィードバック用として使用可能。（例：「満足」または「不満足」に応じて異なる処理へ進む）                                                                                                   |
| **ボタン未クリック時の処理**  | ブール型  | 設定されたボタンにおいて**ユーザーがボタンをクリックせずに、入力欄にテキストを入力して送信した場合**、「未クリックボタン」の値は `true` として反映「未クリックボタン」ノードを次の処理に接続している場合、入力送信後に当処理が実行される                                                                                  |
| **ボタンアクション**      | ブール型  | **ボタンクリック時、テキスト入力がある場合**：ボタンに紐づく処理の実行とともに入力内容も送信**ボタンクリック時、テキスト入力がない場合**：ボタンに紐づく処理のみが実行される                                                                                                                  |
| **モジュール実行完了**     | ブール型  | **本モジュールの実行完了時、出力値は `True` となる**（通常、次処理を開始するためのトリガーとして使用）                                                                                                                                                   |
